# TIB.TJ — MVP

Готовый сайт-навигатор: главная страница, список врачей/клиник с фильтром по специальности, запись через WhatsApp с готовым текстом, переключатель RU/TJ. Данные пока тестовые (`lib/data.js`) — реальных врачей и клиники добавишь позже, когда будут первые партнёры.

## Что нужно сделать, чтобы сайт заработал на tib.tj

### Шаг 1. Залить код на GitHub
1. Зарегистрируйся на [github.com](https://github.com) (если аккаунта нет).
2. Создай новый репозиторий: кнопка **New** → назови `tib-tj` → **Create repository**.
3. На странице пустого репозитория нажми **uploading an existing file** и перетащи в окно браузера всю папку `tib-tj` целиком (структура папок сохранится). Закоммить (**Commit changes**).

### Шаг 2. Задеплоить на Vercel
1. Зайди на [vercel.com](https://vercel.com) → **Sign up** → выбери "Continue with GitHub" (проще всего, аккаунты свяжутся автоматически).
2. **Add New → Project** → выбери репозиторий `tib-tj` → Vercel сам определит, что это Next.js.
3. Нажми **Deploy**. Через 1-2 минуты получишь рабочую ссылку вида `tib-tj.vercel.app` — сайт уже живой.

### Шаг 3. Подключить домен tib.tj
1. В проекте на Vercel: **Settings → Domains → Add** → введи `tib.tj`.
2. Vercel покажет, какие DNS-записи добавить (обычно A-запись или CNAME).
3. Зайди в панель управления доменом на [ism.tj](https://ism.tj) (регистратор) → раздел DNS → добавь записи, которые показал Vercel.
4. Подожди 10-60 минут — домен подключится, Vercel сам выпустит SSL-сертификат (https).

### Шаг 4 (позже). Подключить Supabase — реальные врачи/клиники вместо тестовых данных
Пока сайт работает на данных из `lib/data.js` — этого достаточно для показа концепции первым партнёрам. Когда будешь готов подключить настоящую базу:
1. Создай проект на [supabase.com](https://supabase.com).
2. В SQL Editor выполни файл `supabase/schema.sql` из этого проекта — он создаст таблицы `clinics`, `doctors`, `specialties`.
3. Скопируй `.env.example` в `.env.local`, впиши `NEXT_PUBLIC_SUPABASE_URL` и `NEXT_PUBLIC_SUPABASE_ANON_KEY` (найдёшь в Supabase → Settings → API).
4. Напиши Claude "подключи Supabase к сайту" — код в `lib/data.js` заменится на запросы к базе.

## Перед показом партнёрам — обязательно замени
В файле `lib/data.js` номера WhatsApp сейчас тестовые (`992000000001` и т.д.) — замени на реальные номера клиник, иначе кнопка записи будет вести в никуда.

## Структура проекта
- `app/` — страницы (главная, `/doctors`, `/clinics`)
- `components/` — переиспользуемые блоки (шапка, карточка врача, кнопка WhatsApp, переключатель языка)
- `lib/data.js` — тестовые данные врачей/клиник/специальностей
- `lib/i18n.js` — тексты на русском и таджикском
- `supabase/schema.sql` — схема базы данных для этапа "Пилотные клиники"
